# 课程大实践项目

面向《基于模型的系统工程》课程大实践的独立软件项目语境。它不等同于前 12 个指导书实验，而是以 MBSE / SysML 活动为对象，产出一个可演示、可说明工程价值的软件系统。

## Language

**课程大实践项目**:
独立于前 12 个指导书实验的综合软件作品；评估重点是它如何支持 MBSE 建模、分析、追溯或生成流程。
_Avoid_: 项目 1-12 子实验、普通课程报告

**MBSE 建模工作台**:
以 MBSE 任务为中心的桌面应用，允许用户输入需求材料、查看模型工件，并让建模 Agent 生成、检查或修正这些工件。
_Avoid_: 纯图浏览器、纯 Agent 控制台、Sysbuilder 自动化脚本集合

**SysML 前端展示**:
将 SysML / MBSE 模型中的结构、需求、行为、追溯等视图以可交互方式呈现给用户的产品能力。
_Avoid_: 静态截图展示、纯报告插图

**SysML v2 文本**:
作为 MBSE 建模工作台的权威模型表达，用于保存需求、结构、行为与追溯等模型语义，并作为可提交、可解释的标准化工件。
_Avoid_: 只供展示的伪代码、不可解析的图说明文本

**JSON 视图模型**:
从 SysML v2 文本派生或与其同步的前端渲染模型，用于表达节点、关系、矩阵、布局和校验结果。
_Avoid_: 独立业务真相源、脱离 SysML 的自有模型格式

**建模 Agent**:
面向系统工程任务生成、检查、解释或改写模型内容的代理能力。
_Avoid_: 泛用聊天机器人、只回答概念问题的问答助手

**Agent Sidecar**:
由 Tauri 启动并管理的本地独立进程，承载建模 Agent 会话、模型调用、工具执行和事件流。
_Avoid_: 前端内嵌 Agent、不可控远程服务、一次性 CLI 包装

**SDK 集成边界**:
Agent Sidecar 直接使用 pi / oh-my-pi SDK 管理会话，并向 Tauri 暴露本项目自定义的模型工件与流式事件接口。
_Avoid_: 把 ACP 当成 Agent 能力本身、把 stdout 文本解析当成稳定协议

**需求到多视图闭环**:
从用户输入的任务或需求材料出发，由建模 Agent 生成 SysML v2 文本与 JSON 视图模型，并在工作台中展示需求、结构、行为和追溯视图的端到端能力。
_Avoid_: 单图生成、单次问答、没有模型源的报告生成

**天问二号探测器案例**:
MBSE 建模工作台的默认演示系统，用于从使命与需求材料生成探测器需求、分系统结构、行为流程和追溯矩阵。
_Avoid_: 泛化航天器案例、与课程无关的虚构系统

**材料导入 + 向导确认**:
用户先导入或粘贴天问二号任务/需求材料，建模 Agent 抽取候选使命、约束、需求、分系统和关系，再由用户在向导中确认后生成模型工件。
_Avoid_: 完全自由生成、完全手填表单、一键模板演示

**四视图 + IBD/参数**:
需求到多视图闭环的扩展视图集合，包括需求视图、BDD、活动图、追溯矩阵、IBD 和参数约束视图。
_Avoid_: 完整 Sysbuilder 图编辑器、无边界的工业级建模套件

**可视化 + 静态校验**:
IBD 与参数视图只负责展示部件、端口、连接、约束、参数绑定和单位，并检查引用、覆盖、连接和参数缺失等一致性问题。
_Avoid_: 拖拽式完整图编辑、参数求解仿真、Sysbuilder/Sysplorer 工程导出

**React Flow + elkjs**:
MBSE 建模工作台的前端图形渲染栈；React Flow 负责节点、边、端口和交互，elkjs 负责自动布局。
_Avoid_: 专业建模框架优先、自研图编辑器、把 ELK 当成渲染框架

**单 Agent + 确定性校验**:
由一个建模 Agent 完成材料抽取、SysML v2 生成和修正建议，所有关键输出再通过 schema、引用、一致性和覆盖规则校验。
_Avoid_: 多 Agent 复杂编排、无校验自由生成、只靠 LLM 自评

**完整课程交付包**:
课程大实践项目的最终提交边界，包括源码、可运行 Tauri 应用、天问二号样例工程、SysML v2 与 JSON 模型工件、截图、演示说明和课程报告。
_Avoid_: 只交应用、只交模型文件、只交研究文档

**模型工件**:
课程大实践系统读取、生成或展示的工程对象，例如 SysML 文本、需求条目、结构关系、行为关系、追溯矩阵或视图数据。
_Avoid_: 普通文件、附件
